第一阶段：基础设施层（从启动到路由）
第一步 ：启动入口 main.go

配置加载、数据库连接、Redis/RabbitMQ连接、服务器启动
第二步：路由层 internal/http/router.go

如何将URL路径映射到具体处理函数
中间件链的执行顺序（JWT鉴权、软鉴权等）
第三步：配置层 internal/config/loadconfig.go + internal/db/db.go

配置文件如何被解析为Go结构体
数据库连接如何建立、AutoMigrate如何自动建表

第二阶段：中间件层（鉴权、缓存、消息队列）
第四步：Redis中间件 internal/middleware/redis/redis.go

Redis连接、Get/Set/Del、分布式锁等操作封装
第五步：RabbitMQ中间件 internal/middleware/rabbitmq/rabbitMQ.go

消息队列连接、消息发布/消费逻辑
第六步：JWT鉴权中间件 internal/middleware/jwt/ + internal/auth/jwt.go

JWT Token的生成、验证、硬鉴权/软鉴权区别

第三阶段：核心业务模块（逐模块深入）
第七步：用户模块 internal/account/（handler + service + repo + entity）

注册、登录、修改密码、改名、登出
第八步：视频模块 internal/video/（video_handler + video_service + video_repo + video_entity）

发布视频、查询作者视频列表、视频详情（含缓存逻辑）
第九步：点赞模块 internal/video/like_*（handler + service + repo + entity）

点赞/取消点赞、判断是否已点赞（含MQ异步处理逻辑）
第十步：评论模块 internal/video/comment_*（handler + service + repo + entity）

发布评论、删除评论、查询视频评论列表（含MQ异步处理逻辑）
第十一步：关注模块 internal/social/（handler + service + repo + entity）

关注/取关、查询粉丝列表、查询关注列表（含MQ异步处理逻辑）
第十二步：Feed流模块 internal/feed/（handler + service + repo + entity）

最新视频流、按点赞数排序、按热度排序、关注者视频流（含复杂缓存和游标分页逻辑）
第四阶段：异步Worker层（后台任务处理）
第十三步：Worker入口 cmd/worker/（如果存在）.

Worker如何独立运行、如何消费RabbitMQ消息
第十四步：点赞Worker internal/worker/likeworker.go

异步处理点赞/取消点赞事件，更新MySQL计数和Redis缓存
第十五步：评论Worker internal/worker/commentworker.go

异步处理评论发布/删除事件
第十六步：热度Worker internal/worker/popularityworker.go

异步处理视频热度更新，维护Redis热榜
第十七步：关注Worker internal/worker/socialworker.go

异步处理关注/取关事件
